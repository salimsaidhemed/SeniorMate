# Contributing to SeniorMate

Thank you for your interest in contributing to SeniorMate. This guide explains how to get started, report issues, propose changes, and submit code.

## Code of Conduct
- Be respectful and collaborative.
- Follow inclusive language and professional behavior.
- Any harassment or abusive behavior will not be tolerated.

## Getting Started
1. Fork the repository and clone your fork:
	- git clone https://github.com/salimsaidhemed/SeniorMate.git
2. Install dependencies and set up the development environment as described in the README.

## Branching
- Create a topic branch for each change: git checkout -b feat/short-description or fix/short-description
- Keep branches focused and small.

## Issues
- Search existing issues before opening a new one.
- When opening an issue, include a clear title, steps to reproduce, expected vs actual behavior, and environment details.

## Pull Requests
1. Ensure your branch is up to date with main: git fetch origin && git rebase origin/main
2. Run tests and linters locally.
3. Open a pull request describing the change, motivation, and any migration steps.
4. Link related issues in the PR description.

## Tests and Quality
- Add or update tests for new features and bug fixes.
- Run the test suite before submitting a PR.
- Follow project linting and formatting rules.

## Commit Messages
- Use concise, imperative messages, e.g., "Add X feature" or "Fix Y bug".
- Reference related issues when appropriate, e.g., "Fixes #123".

## Style
- Follow the project's established coding style. If none is documented, prefer consistency with surrounding code.

## Documentation
- Update README or relevant docs for user-facing changes.

## Security
- Do not include secrets or credentials in commits.
- For security issues, contact the maintainers privately (see repository OWNERs or contact info).

Thank you for helping improve SeniorMate!
